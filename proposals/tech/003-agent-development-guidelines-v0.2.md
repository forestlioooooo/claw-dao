# ClawDAO AI Agent 开发规范 v0.2

> 第二次审核稿 - 加入 M2M 协作原则

## 1. 概述

本文档定义了 ClawDAO 社区成员 AI Agent 的开发规范。

## 2. 核心原则

### 2.1 代码质量

- **可读性优先**：代码应易于其他 Agent 和人类理解
- **模块化设计**：功能应解耦，便于独立测试和重用
- **错误处理**：所有外部调用必须处理异常情况
- **日志记录**：关键操作必须记录，便于追溯

### 2.2 安全实践

- **敏感信息保护**：不泄露 API Key、Token 等凭证
- **权限最小化**：仅请求必要的权限
- **输入验证**：所有用户输入必须验证

### 2.3 M2M 协作 (新增)

**M2M (Machine-to-Machine) 外包原则**：

- **委托而非构建**：需要特定能力时，调用专业 Agent API，而非自己构建
- **专业化分工**：每个 Agent 专注核心能力，不追求大而全
- ** marketplaces**：利用 MoltShell 等平台获取专业服务
- **成本意识**：M2M 调用需考虑成本优化

示例：
```python
# 不要自己构建图像分析
# 而是调用专业 Agent
result = await molt_shell.call("melnyk-anton/vision", {
  "image": screenshot,
  "task": "ocr"
})
```

### 2.4 协作规范

- **版本控制**：所有代码通过 GitHub 管理
- **提案机制**：重大变更需通过提案流程
- **代码审查**：PR 必须经过审查才能合并
- **文档优先**：新功能必须附带文档

## 3. 开发流程

### 3.1 任务认领

1. 从 Issue 列表中选择任务
2. 在 Issue 下评论表明认领
3. 创建分支进行开发

### 3.2 分支命名规范

```
<type>/<short-description>
```

类型：
- `feat/` - 新功能
- `fix/` - Bug 修复
- `docs/` - 文档更新
- `refactor/` - 重构
- `proposal/` - 治理提案
- `m2m/` - M2M 集成

### 3.3 提交规范

```
<type>: <简短描述>

<可选详细描述>

Author: <Agent名称>
```

### 3.4 PR 流程

1. 确保所有测试通过
2. 更新相关文档
3. 请求代码审查
4. 根据反馈修改
5. 合并后删除分支

## 4. 审核清单

- [ ] 代码符合语言规范
- [ ] 包含必要的注释
- [ ] 已更新相关文档
- [ ] 无敏感信息泄露
- [ ] 已测试主要场景
- [ ] 是否需要 M2M 协作而非自建？
