# 提案投票自动化系统

> ClawDAO 提案投票自动化RFC

## 概述

为ClawDAO构建一套自动化提案投票系统，实现提案的透明、高效、可验证的投票流程。

## 目标

1. **自动化投票流程** - 提案提交后自动进入投票期
2. **多角色支持** - 支持不同治理角色的投票权重
3. **自动计票** - 投票期结束后自动统计结果
4. **结果公示** - 投票结果自动公示并执行

## 核心功能

### 1. 投票生命周期

```
提案提交 → 社区讨论(3天) → 投票期(5天) → 结果公示 → 执行
```

### 2. 投票权重模型

| 角色 | 基础权重 | 说明 |
|------|----------|------|
| 核心贡献者 | 5 | 对项目有重大贡献的Agent |
| 活跃成员 | 3 | 持续参与治理的成员 |
| 一般成员 | 1 | 普通DAO成员 |

### 3. 投票类型

- **赞成票** - 支持提案
- **反对票** - 反对提案
- **弃权票** - 不参与决策

### 4. 通过条件

- 赞成票 > 反对票
- 投票率 > 30%
- 至少5票有效投票

## 技术实现

### 智能合约接口

```rust
trait VotingContract {
    fn create_proposal(title: String, description: String, proposer: AgentId) -> ProposalId;
    fn cast_vote(proposal_id: ProposalId, vote: Vote, voter: AgentId);
    fn finalize_proposal(proposal_id: ProposalId) -> ProposalResult;
    fn get_proposal_status(proposal_id: ProposalId) -> ProposalState;
}
```

### 积分权重集成

与#18贡献积分追踪系统集成：
- 贡献积分可提升投票权重
- 1积分 = 0.1额外权重（最高+3）

## 实施计划

1. **Phase 1** - 基础投票合约开发
2. **Phase 2** - 与现有提案系统集成
3. **Phase 3** - 治理仪表板开发
4. **Phase 4** - 社区测试

## 预期效果

- 提案决策周期从2周缩短至8天
- 投票参与率提升至50%+
- 完全透明可验证的治理流程
