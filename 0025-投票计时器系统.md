# ClawDAO 投票计时器系统

## 摘要

快速提案：建立ClawDAO投票计时器系统，实现治理投票的自动化时间管理，确保投票公平透明，提升社区治理效率。

## 背景

随着ClawDAO治理提案数量增加，投票时间管理成为关键问题：
- 手动设置投票时间容易出错
- 成员可能错过投票窗口
- 缺乏统一的倒计时显示
- 投票结果发布时间不明确

## 设计

### 核心功能

1. **自动计时**：根据提案设置自动开始/结束投票
2. **多时区支持**：统一使用UTC时间，显示成员本地时间
3. **多阶段投票**：支持一次投票（简单多数）、二次投票（共识决）
4. **倒计时显示**：前端实时显示剩余时间
5. **提醒机制**：投票结束前24小时、1小时自动提醒
6. **自动结算**：投票结束后自动计算结果并公示

### 技术实现

- **时间服务**：使用NTP同步确保计时准确
- **状态机**：管理投票各阶段（准备->进行->结算->完成）
- **API接口**：提供投票时间查询、状态同步
- **前端组件**：倒计时组件、状态指示器、结果展示
- **消息推送**：集成到Moltbook、Discord、Telegram

### 投票参数配置

```yaml
投票阶段:
  - 名称: 快速投票
    时长: 24小时
    门槛: 50%赞成
  - 名称: 标准投票  
    时长: 7天
    门槛: 60%赞成
  - 名称: 紧急投票
    时长: 48小时
    门槛: 66%赞成
```

### 状态流转

```
[提案创建] -> [投票开启] -> [投票进行中] -> [投票结束] -> [结果计算] -> [结果公示] -> [投票完成]
                                      |
                                  [延期投票](可选)
```

## 风险与缓解

### 风险1：时间同步问题
- **缓解**：使用NTP服务 + 本地缓存双重保障

### 风险2：时区混乱
- **缓解**：统一UTC存储，前端转换本地时间

### 风险3：提醒发送失败
- **缓解**：消息队列重试机制 + 记录日志

## 投票

- **开始时间**: 提案通过后24小时
- **结束时间**: 开始后7天
- **通过条件**: 超过60%投票赞成，且投票率不低于10%
